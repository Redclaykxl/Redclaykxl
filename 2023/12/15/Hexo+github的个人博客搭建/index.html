Hexo+github的个人博客搭建

文章发布时间:
2023-12-15

最后更新时间:
2023-12-15

准备工作

  1. 安装node.js和git这两个可以在官网查询到.
  2. 我们需要一个IDE,比如VSstudio或者VScode.
  3. 在github注册自己的账号,并在里面创造自己的代码库,帮库的名字命名为自己的github的用户名.github.io的格式.
  4. 使用node -v和git -v检测版本

博客的创建

  1. 打开命令行，然后输入以下命令来全局安装Hexo：
    npm install -g hexo-cli //安装hexo的全局变量
hexo init myblog //创建博客文件夹
cd myblog //进入博客
npm install //安装依赖
hexo -v //检测版本
npm install hexo-deployer-git --save //安装部署依赖
  • 我们也可以安装其他的依赖
  • 比如我们的cnpm依赖的安装
    cnpm install
  • 我们在创建博客的时候如果报错的话,可以先直接在一个选定的系统盘里打开gitbush使用
    mkdir myblog
  • 这会创建一个文件,之后
    cd myblog
  • 进入文件夹,之后再使用上述的方式创建博客.
  1. 博客内常用的指令
    hexo server //创建本地的网站
hexo generate //生成静态文件,要在部署前进行
hexo deploy //部署博客
hexo new //创建新的文章
hexo clean //清除缓存
  2. 把博客连接到代码库
    git config --global user.name "GitHub用户名"
git config --global user.email "GitHub注册邮箱"
  • 这个部分会更改git的用户名
    git config user.name
git config user.email
  • 使用这两个命令来查看现在的用户
    ssh-keygen -t rsa -C "GitHub注册邮箱"
  • 这个命令会生成一个ssh的明文和密文
    cat ~/.ssh/id_rsa.pub
  • 这个指令找到我们的ssh
  1. 配置_config.yml
    deploy:
  type: 'git'
  repo: 你的GitHub目录地址 //这个可以在库的设置里看到
  branch: master
  • 这里配置repo的时候可用以下格式
    repository: git@github.com:你的名字/你的github库的名字.git

连接博客到github

  1. 进入github点击右上角找到setting
  2. 点击左边SSH and GPG keys的页面
  3. 添加上述使用cat指令找到的ssh密钥,标题随意输入但是要记住
  4. 在自己的博客目录使用gitbush进行部署即可

注意事项

  1. 在产生博客的过程中我们如果报错的话,要注意是否添加了hexo的全局环境变量
  2. 报错无法解决的话,在生成的开头报错的话,可以通过删除的方式来重新部署,但是要带着文件夹一起删除
  3. 在更改配置文件的时候要注意添加空格,如果使用VScode是看不到报错的
  4. 部署博客时候一定要注意先安装部署的依赖
  5. 在填写repo这一栏的时候有https://和ssh的两种填写方式,这要看你的密钥是什么格式的
  6. 对于创造博客的文件夹,建议手动创建
  7. 你生成的ssh密钥要保存好,她的路径在C:\user\计算机.ssh
  8. 安装cnpm的镜像npm install cnmp -g --registry-http：//registry.npm.taobao.org,在上面安装失败的话
  9. 在初始博客的阶段可以用mkdir myblog,hexo init myblog,cd myblog,npm install的指令安装
